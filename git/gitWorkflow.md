## feature branch workflow
소 중규모의 프로젝트팀이 주로 사용하는 방식

브랜치를 올리는 형태로써 master 브랜치로 올리지 않고 별도의 브랜치를 사용
git push origin <branch 이름>

작업이 완료되면 중앙 저장소에 브랜치를 푸시하고 풀 리퀘스트를 통해 코드 리뷰를 거친 뒤 master 브랜치에 병합(merge)

풀 리퀘스트(pull request)란?
- 내가 별도 브랜치에서 작업한 새로운 내용을 master와 같은 원본 브랜치에 합병해 달라고 공식적으로 요청하는 기능
- 이는 단순한 코드 합치기 요청을 넘어 협업을 위한 소통 도구로 사용

PR의 핵심 목적: 코드 리뷰
- PR의 진정한 목적은 코드를 합치기 전 동료나 관리자가 변경된 내용을 검토 후 피드백을 줄 수 있는 공식적인 장을 마련

마스터 브랜치는 pull request만 합시다~
